— FAQ

Common questions.

Why are these free? What's the catch? +
No catch. This is the launch promo. We're a new studio and we'd rather have 1,000 servers running our scripts and giving us feedback than 50 servers paying for v1. After the launch period, new scripts will be paid — but everything currently free will stay free for early adopters, with all updates included.
What framework do these target? +
QBox primary, QBCore as a fallback via auto-detection. All scripts use ox_lib + oxmysql universally and ox_inventory where item handling is needed.
How do I install them? +
For each script: clone the GitHub repo into your resources/ folder, import the SQL from sql/install.sql, add an ensure line to your server.cfg, restart. Each repo's README has the 5-step walkthrough.
Is the source obfuscated? +
No. Everything is plain Lua, well-commented, public on GitHub. Audit it, extend it, modify it for your server.
Do you offer support? +
Yes. GitHub issues for bug reports (tracked publicly), Discord for live chat. Bug fixes typically ship within 24 hours of being reported.
Can I modify them for my server? +
Yes, modify freely for your own server. Single-server use license — redistribution or reselling is not permitted. Lifetime updates included for everyone who downloaded during the launch period.
How do the scripts integrate with each other? +
Every script exposes exports that other resources can call. Reputation exposes AddReputation, etc. Events optionally grants criminal rep on wins. DOJ optionally uses reputation tier for bail and sentencing. Businesses optionally applies legal-rep-tier discounts. Each integration is wrapped in pcall — missing dependencies never crash anything. They work standalone or as a system.
Are you available for custom development? +
Yes — for serious servers. Drop us a line with what you're building and we'll discuss scope, timeline, and rates.
